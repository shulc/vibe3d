## План: улучшение edge bevel в vibe3d

### Контекст

Два взаимосвязанных дефекта в `applyEdgeBevelTopology`:

1. **Направления скольжения** вычисляются как `normalize(neighbor - pos)` — зависят от длины соседнего ребра, не дают постоянного перпендикулярного расстояния `ebWidth` от исходного ребра.
2. **Стыки (junction vertices)** обрабатываются усреднением направлений — все граничные вершины стыка попадают в одну точку вместо образования полигональной шапки.

Оба дефекта исправляются последовательно; Phase 2 опирается на Phase 1.

---

### Phase 1 — Перпендикулярный offset (замена направлений)

**Идея** (из Blender `offset_meet`):  
Для вершины `va` на ребре `(va, vb)`, в грани F1 = `[…, prevA, va, vb, …]`:

```
dir1 = normalize(va - prevA)       // ребро prevA→va, входит в va
dir2 = normalize(vb - va)          // ребро va→vb, выходит из va
norm_v = normalize(cross(dir2, dir1))   // нормаль в плоскости F1 у va
           // ориентировать dot(norm_v, faceNormal(F1)) > 0
norm_perp1 = normalize(cross(dir1, norm_v))  // перпендикуляр к ребру 1
norm_perp2 = normalize(cross(dir2, norm_v))  // перпендикуляр к ребру 2

off1a = va + norm_perp1 * 1.0   // offset-линия ребра 1 при width=1
off1b = off1a + dir1
off2a = va + norm_perp2 * 1.0
off2b = off2a + dir2

meetA1 = линейное пересечение off1a–off1b и off2a–off2b
dirA1  = meetA1 - va            // сохраняем как direction (lineal in width)
```

Аналогично для A2 (в грани F2), B1 и B2.  
`updateEdgeBevelVertices` не меняется — `origA + dirA1 * ebWidth` по-прежнему корректно.

**Специальные случаи** (как в Blender):
- Параллельные рёбра (угол < ε): `dirA1 = norm_perp * 1/cos(ang/2)`
- Антипараллельные (угол ≈ π): `dirA1 = norm_perp1 * 1.0` (скользим вдоль одного перпендикуляра)

**Изменения в коде:**

1. **`math.d`** — добавить две функции:
   - `Vec3 faceNormal(const uint[] face, Vec3[] verts)` — нормаль грани по вершинам
   - `Vec3 lineMeetOffset(Vec3 va, Vec3 dir1, Vec3 dir2, Vec3 faceNorm, float width)` — вычисляет позицию пересечения offset-линий двух рёбер при заданном width (возвращает точку, из которой вычитается `va` для получения direction)

2. **`bevel.d` строки 772–775** — заменить 4 строки `safeNorm(neighbor - pos)` на вызовы `lineMeetOffset`

### Phase 2 — Полигональная шапка на стыках (junction cap)

**Идея:**  
На стыковой вершине V с N выбранных рёбер (в угловом порядке E1…EN):
- Между каждыми двумя соседними рёбрами Ei и Ei+1 есть «зазорная грань» (gap face) — грань, содержащая и Ei, и Ei+1 у вершины V
- Пересечение offset-линий Ei и Ei+1 в этой грани даёт одну точку — **junction gap vertex** Pi
- Это значит: `nvA2_Ei` и `nvA1_Ei+1` должны быть **одной и той же вершиной** (один индекс), а не двумя совпадающими
- Из N таких точек строится N-угольная шапка

**Изменения в коде:**

1. **Новая предобработка** (перед циклом по рёбрам, строка ~756):
   ```d
   // Для каждой junction-вершины V: обойти рёбра V в угловом порядке через грани,
   // для каждого зазора между соседними выбранными рёбрами создать одну вершину.
   int[ulong] jGapVert;  // key = (junctionV << 32 | gapFaceIdx) → new vert idx
   ```
   - Обход: начиная с любого выбранного ребра Ei у V, взять F1(Ei), найти предыдущее ребро V в этой грани, перейти на соседнюю грань — и так по кругу пока не найдём следующее выбранное ребро
   - Для каждого зазора: `addVertex(va_pos)` → записать в `jGapVert`
   - Сохранить угловой порядок для создания шапки

2. **Модификация цикла создания вершин** (строки 778–794):
   - Для endpoint на junction-вершине: вместо `addVertex` для nvA1/nvA2 — брать из `jGapVert` для соответствующего зазора
   - Для non-junction endpoint: без изменений (по-прежнему 2 свежих вершины)

3. **Удалить блок усреднения направлений** (строки 846–886) — весь блок `if (junctionVerts.length > 0) { ... }`

4. **Добавить создание шапок** (после строки 918, после добавления bevel-квадов):
   ```d
   foreach (V, gapVerts; junctionCapVerts) {
       if (gapVerts.length >= 3)
           mesh.faces ~= gapVerts;  // N-угольник
   }
   ```

5. **Корректировка таблицы замен** (строки 807–838):
   - Зазорная грань теперь получает ровно ОДНУ замену V (shared gap vertex) — а не две как сейчас
   - Это автоматически следует из sharing vertex index

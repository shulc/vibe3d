# Алгоритм Edge Bevel в Blender

Источник: `blender-main/source/blender/bmesh/tools/bmesh_bevel.cc` (8157 строк)

---

## Основные структуры данных

**`BevVert`** — вершина, которая бевелируется. Хранит массив `EdgeHalf` (рёбра вокруг вершины в CCW-порядке), `VMesh` (новая сетка вместо вершины), `selcount` (сколько рёбер бевелируются).

**`EdgeHalf`** — один конец ребра в вершине. Ключевые поля:
- `offset_l`, `offset_r` — смещения левой и правой стороны
- `leftv`, `rightv` — граничные вершины (`BoundVert`)
- `fprev`, `fnext` — соседние грани
- `is_bev` — бевелируется ли ребро
- `is_rev` — перевёрнуто ли направление ребра

**`BoundVert`** — вершина на границе нового vertex mesh. Образуют круговой список вокруг вершины. У каждой есть `Profile` — форма бевела до следующей `BoundVert`.

**`Profile`** — кривая (сечение бевела): контрольные точки `start/middle/end`, параметр `super_r` (суперэллипс), массив `prof_co` — готовые точки на кривой.

**`VMesh`** — новая сетка вместо вершины. Массив `NewVert[count × (seg/2+1) × (seg+1)]`. Тип:
- `M_POLY` — один n-угольник
- `M_ADJ` — сетка через подразбиение (основной случай)
- `M_TRI_FAN` — треугольный веер
- `M_CUTOFF` — срез по профилю

---

## Пайплайн (порядок вызовов в `BM_mesh_bevel`)

```
1. bevel_vert_construct()     — построить BevVert для каждой затронутой вершины
2. build_boundary()           — построить BoundVert-ы и определить форму границы
3. adjust_offsets()           — выровнять ширины (least-squares, если нужно)
4. calculate_vm_profiles()    — посчитать точки профилей
5. build_vmesh()              — построить внутренние вершины и BMVert-ы
6. bevel_rebuild_polygon()    — перестроить исходные грани
7. bevel_reattach_wires()     — переподключить wire-рёбра
```

---

## Шаг 1 — Построение BevVert

- Для каждой бевелируемой вершины собираются все инцидентные рёбра в CCW-порядке (`find_bevel_edge_order`).
- Вычисляются `offset_l_spec` / `offset_r_spec` для каждого `EdgeHalf` в зависимости от режима:
  - **OFFSET** — прямое смещение
  - **WIDTH** — `offset = width / (2 × sin(dihedral/2))`
  - **DEPTH** — `offset = depth / cos(dihedral/2)`
  - **PERCENT** — `offset = edge_length × pct / 100`

---

## Шаг 2 — Построение границы (`build_boundary`)

Для каждого бевелируемого ребра нужно найти, где его «полоса» встречается с соседними:

**Нормальный случай** (`offset_meet`):
1. Берём два смежных `EdgeHalf` и их offset-плоскости
2. Пересекаем сдвинутые прямые — получаем позицию `BoundVert`
3. Обрабатываем вырожденные случаи: параллельные рёбра, антипараллельные

**Если между бевелируемыми рёбрами есть небевелируемые:**
- Если `loop_slide` включён → `BoundVert` скользит по промежуточному ребру (`eon`)
- Иначе → вычисляется по граням

**Специальные случаи:**
- **Terminal edge** (одно бевелируемое ребро в вершине): создаются 2–3 `BoundVert` вдоль небевелируемых рёбер
- **Vertex bevel**: по одной `BoundVert` на каждое ребро
- **Miters** (рефлекс/concave): создаются 1–3 дополнительных `BoundVert` для скруглённого или прямого стыка

После этого определяется `vmesh_kind`:
- 2 бевелируемых ребра, 2 `BoundVert` → weld (нет VMesh, просто соединить)
- 1 бевелируемое → `M_TRI_FAN`
- Общий случай → `M_ADJ`
- `seg=1` или плоский → `M_POLY`

---

## Шаг 3 — Выравнивание offset-ов

Если `offset_adjust=true` или есть `BoundVert` на рёбрах (`eon`), то:
- Строятся цепочки/циклы зависимых `BoundVert`
- Решается задача наименьших квадратов: минимизировать отклонение от spec при равных реальных ширинах
- Граница перестраивается с новыми значениями

---

## Шаг 4 — Профили (`calculate_vm_profiles`)

Для каждой `BoundVert` → `calculate_profile()`:

1. Плоскость профиля задаётся нормалью ребра и точками `start`/`end`/`middle`
2. Строится отображение единичного квадрата [0,1]² → 3D (`make_unit_square_map`)
3. Сэмплируются `seg+1` точек:
   - **Суперэллипс**: `(|x|^r + |y|^r)^(1/r) = 1`, r=2 → окружность, r=1 → прямая
   - **Пользовательский профиль**: из `CurveProfile`
4. Результат в `prof_co[]` и `prof_co_2[]` (для subdivision)

---

## Шаг 5 — Построение VMesh (`build_vmesh`)

**Weld case** (2 ребра, selcount=2):
- Нет внутренней сетки
- Профили двух `BoundVert` совмещаются: берётся одна кривая, вторая — зеркало
- `move_weld_profile_planes()` выравнивает плоскость по исходной вершине

**M_ADJ** (общий случай, `adj_vmesh`):
1. Строится грубая сетка при `seg=2`
2. Вычисляется центральная вершина через «полноту» (fullness) профиля
3. Итеративное Catmull-Clark subdivision вдвое, пока не достигнем нужного `seg`
4. Если `seg` не степень двойки → финальная интерполяция
5. Специальные оптимизации: трубы (два коллинеарных ребра), углы куба, квадратные профили

**M_TRI_FAN**:
- Один полигон, разделённый треугольниками из центральной вершины

**M_CUTOFF**:
- Профильные грани + нижняя грань

После вычисления координат создаются реальные `BMVert`.

---

## Шаг 6 — Создание рёбер и граней (`bevel_build_rings`, `bevel_build_polygons`)

**Кольцевые грани** вдоль бевела:
- Квады `(i,j,k)–(i,j,k+1)–(i,j+1,k+1)–(i,j+1,k)` из сетки VMesh
- UV интерполируется из соседних оригинальных граней

**Перестройка исходных граней** (`bev_rebuild_polygon`):
- Обход оригинальной грани; в каждой затронутой вершине вставляются `BoundVert`-ы с их профильными точками
- Создаётся новая грань; атрибуты интерполируются из оригинала
- Проверяется continuity швов (seam) и sharp-рёбер

---

## Ключевые частные случаи

| Случай | Обработка |
|--------|-----------|
| 2-edge weld | Нет VMesh, профили сшиваются напрямую |
| Terminal edge | BoundVert-ы на небевелируемых рёбрах, M_TRI_FAN |
| Pipe (два коллинеарных ребра) | Snap профилей к цилиндру |
| Cube corner (3 ребра под 90°) | Специальная симметричная сетка |
| Non-manifold edge | Пропускается |
| Wire edges | Сохраняются отдельно, переподключаются к ближайшему BoundVert после построения |
| limit_offset | Уменьшает offset, если ширина вызвала бы инверсию |
